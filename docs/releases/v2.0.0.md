## 릴리스 목적

V1의 발음 휴리스틱을 버리는 대신 deterministic candidate retrieval로 제한하고, 문맥상 모호한 후보 선택만 Microsoft Foundry 또는 Ollama에 맡기는 bounded agent 구조로 전환합니다. 모델은 자유로운 replacement 문자열을 만들지 못하며 로컬 validator가 span, candidate ID, confidence와 DB vocabulary를 다시 검사합니다.

## 포함 범위

- Microsoft Foundry Project Responses API를 기본 provider로 사용.
- Entra ID/Managed Identity 인증; OpenAI API key 불필요.
- Ollama를 명시적 로컬 옵션으로 제공.
- OpenAI·Claude는 reference-only이며 adapter와 credential을 요구하지 않음.
- V1 `PronunciationMapper` API와 CLI 호환 유지.
- conservative number normalization, particle/identifier/punctuation 보존.
- provider trace와 fallback 진단이 포함된 typed result 계약.
- 실행 commit, 버전, 입력 hash와 gate를 기록하는 offline/live golden-set evaluation.
- Python 3.10–3.13 CI, wheel build, Pages 정적 검증.
- SPDX/PEP 639 기반 MIT license package metadata.
- external-tenant Foundry OIDC live workflow와 임시 배포 삭제 guard.

## 호환성과 마이그레이션

- V1 호출자는 코드 변경 없이 계속 사용할 수 있습니다.
- V2는 Python 3.10 이상이 필요합니다.
- Azure 사용자는 `.[foundry]`, Ollama 사용자는 `.[ollama]` extra를 설치합니다.
- 운영 전 production query를 golden set으로 고정하고 V2를 shadow mode로 비교하는 절차를 권장합니다.
- 상세 절차는 [V2 아키텍처의 마이그레이션 절](../V2_ARCHITECTURE.md#7-v1에서-마이그레이션)을 따릅니다.

## 알려진 제약

- 대규모 vocabulary는 아직 선형 후보 ranking이며 BK-tree/ANN 인덱스가 필요합니다.
- Ollama 품질은 선택한 모델에 따라 달라지며 production model을 고정하지 않습니다.
- OpenAI와 Claude provider는 구현하지 않았습니다.
- GitHub OIDC workflow는 코드와 로컬 external-tenant 호출까지 검증했지만, repository의 `foundry-external` Environment와 Entra federated credential은 별도 보안 설정이 필요합니다.
- Application Insights/OpenTelemetry 자동 adapter는 아직 제공하지 않습니다.
